## 搬瓦工"带宽"和"流量"是两个概念

先把这个搞清楚，避免后面操作方向跑偏。

搬瓦工的套餐里，**带宽（Bandwidth Speed）** 指的是网络速度上限，比如 1Gbps、2.5Gbps；**流量（Data Transfer）** 指的是每月可以用多少 GB 的总量，比如 1TB/月。两者单位不同，是两件事。

平时说的"搬瓦工升级带宽"，大多数人实际想解决的是：**流量不够用**，或者**线路速度太慢**。前者是月流量上限的问题，后者是套餐所走线路（CN2、CN2 GIA、软银等）的问题。

理清了这个，才能选对方向。

## 三条路径，根据情况选一条

### 路径一：后台自助升级套餐（最常用）

这是处理"流量不够、配置跟不上"最直接的方式。不需要重新买机器，不需要找客服，数据全保留，IP 不变。

操作步骤如下：

1. 登录搬瓦工后台（用户中心）

2. 点击顶部导航 **Services → My Services**

3. 找到需要升级的 VPS，点击右侧的**齿轮图标 ⚙️**，选择 **Upgrade/Downgrade**

4. 页面会列出当前套餐可以切换的更高档方案，选好后点击 **Upgrade to this plan**

5. 系统自动按剩余时间折算差价，只补差额，不是重新付全款

6. 支持支付宝或 PayPal 付款，付完即时生效

差价计算公式是：**（新套餐价 - 旧套餐价）/ 付款周期 × 剩余天数**。剩余时间越多，差价越接近满额；快到期才升的话，补的其实不多。

付完之后不用重启，内存、硬盘、流量配额全部立刻更新。用 `free -m` 或者 `df -h` 确认一下就好。

### 路径二：购买流量包（临时救急）

如果当月流量快超了，但不想立刻换套餐，搬瓦工支持直接在后台购买额外流量包（Bandwidth Top-up）。

适合偶尔超量、平时正常用的情况。长期月月超的话，这条路性价比不高，不如直接升套餐合算。

### 路径三：换到高速线路套餐（改善线路质量）

这是另一种"升级"的理解——不是流量不够，而是速度不够快，晚高峰丢包严重，想换到 CN2 GIA 或者更好的线路。

这种情况也是通过后台升级套餐实现的，操作流程和路径一完全一致，只是目标套餐不同。比如从 KVM 套餐升到 CN2 GIA-E，线路质量就直接提了一档。
